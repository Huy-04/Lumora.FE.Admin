## Notes

- **Reference**: `categories` feature is the gold standard — do NOT modify it. Use it to verify rule understanding.
- **No dependencies**: All 11 tasks are independent and can be executed in parallel.
- **Pattern-only fixes**: Never change business logic, API calls, data flow, routing, computed logic, or watcher behavior.
- **Fix approach**: Read the categories reference for each rule → compare target feature → align to match.
- Each "Apply fixes" sub-task is where actual code changes happen, informed by the preceding audit sub-tasks.
